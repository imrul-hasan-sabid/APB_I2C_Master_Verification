Coverage Top Level Summary Report (Module or Instance Based)

Lists Total coverage and coverage for each requested coverage type. Key terms:
Total Average of coverage percentages for each coverage type with items scored (CoverGroup Bins based coverage is excluded).
BlockBlocks which when Branch is scored also includes any Branches.
BranchBranches found in if-else/case statements (explicit or implicit), in Verilog ternary assignments, and in VHDL conditional and signal assignments. .
StatementIndividual Statements of every Block.
ExpressionExpression Rows.
ToggleNets fully toggled.
FSMStates, Transitions, and optionally Arcs.
AssertionAssert, Assume and Cover directives in PSL and SVA.
CoverGroup WeightedWeighted CoverGroups coverage calculated as the weighted average of coverage for all CoverGroups below.
CoverGroup BinsRelative CoverGroups coverage in percent calculated as the ratio of Covered Bins and Total Bins.
NameOne of Cumulative, Self, or an Instance or Module name for this row of data.
Cumulative Coverage for this instance and all of its sub instances.
SelfCoverage for this instance without any of its sub instances.

Coverage is displayed in one of the following ways:
62 (153/246)For all but CoverGroup coverage: Relative coverage in percent calculated as the ratio of Covered items and Total items. In parenthesis are Covered items and Total items. To not report absolute numbers generate the report with -noabsolute.
62 (153/245/1)As above, but with one Exception item. They include marks by COM, pragma marks, and manual IGN and COV marks. (Covered/Total/Exception)
62Relative coverage in percent in a report generated with -noabsolute.
57 (5)For CoverGroup Weighted coverage: Relative coverage in percent calculated as the weighted average of coverage for all CoverGroups below. In paranthesis is the cumulative weight of all covergroups below. Each CoverGroup adds a default weight of 1. Weights can be modified in source code, and they are displayed in CoverGroup Details tables.
57 (127/1357)For CoverGroup Bins coverage: Relative coverage in percent calculated as the ratio of Covered Bins and Total Bins. In paranthesis are the number of Covered bins and Total bins.
No ItemsThere were no items of this coverage type scored in simulation.
Not ScoredScoring of this coverage type was disabled at simulation or items were deselected before reporting.

Block Coverage, Detailed Report

The Block Coverage report lists coverage for blocks of code.
If Branch Coverage is scored, then it additionally lists separate coverage numbers for all blocks that also are branches
If Statement Coverage is scored, then it additionally lists separate coverage numbers for statements and it lists the number of statements associated with each block.
indexSequential numbering of blocks within a module or instance to use when marking items. Displayed only when indices have been enabled using "indices -on".
cntNumber of hits of this block.
#StmtsNumber of statements within this block (only when statement coverage is scored).
uncovered block line no.Line number for item. An * indicates item is related to that line, e.g. an implicit statement. For a branch item this column also lists its kind: true part of, false part of, implicit else, a case item of, ternary 1 true etc.

Miscellaneous

Report ContentThe Top Level Summary lists among other things the options and database ICCR used to generate this HTML report. Together these settings determine what is available for viewing. Do 'help report_html' in ICCR for more information.
`include FilesSource line information refers to a line in the primary File name. If the source is in a `include, bind, or vunit file, then the line number is followed by a three-letter index which maps to one of the listed `include files.
SortingMacro execution must be enabled to allow sorting. Click any colored column header to sort.
NavigationUse the browser's Back button to return to previous view.
Use right-mouse-button on links to optionally Open Link in New Window.
In general, item names and coverage numbers link to additional detailed information.
CustomizationDo 'help set_report_html_css_path' in ICCR to learn how to customize your report formatting.
